Dzien 1 — Podstawy UiPath Studio, Zmienne, Excel i DataTable
Modul 1 — Czym jest UiPath Studio?

UiPath Studio to srodowisko programistyczne (IDE) do tworzenia robotow RPA. Robot RPA nasladuje czynnosci uzytkownika — klika, wpisuje tekst, czyta dane z ekranu, przenosi pliki — ale robi to automatycznie, bez przerw i bledow wynikajacych ze zmeczenia.

Najwazniejsze koncepcje na start
  • Workflow — przeplyw pracy, czyli sekwencja krokow, ktore robot wykonuje. Zapisywany jako plik .xaml.
  • Activity (Aktywnosc) — pojedynczy krok w workflow (np. klikniecie przycisku, wpisanie tekstu, otwarcie przegladarki). UiPath ma ponad 500 wbudowanych aktywnosci.
  • Sequence — kontener, w ktorym aktywnosci wykonuja sie liniowo, jedna po drugiej (z gory na dol). To najczesciej uzywany typ kontenera.
  • Project — caly projekt automatyzacji, zawierajacy workflow, zaleznosci, pliki danych.

Tryby projektowania — Modern vs Classic

UiPath Studio oferuje dwa tryby projektowania aktywnosci UI:

  • Modern Design (domyslny od wersji 2021.10+) — uzywa Use Application/Browser, Click, Type Into z nowym silnikiem selektorow (Unified Target). Zalecany dla nowych projektow.
  • Classic Design — starszy tryb z Open Browser, Attach Browser, klasycznymi selektorami XML. Nadal spotykany w istniejacych projektach.
Na szkoleniu uzywamy Modern Design.
Selektory — jak robot "widzi" elementy UI?

Selektor to "adres" elementu na ekranie. Gdy klikasz przycisk na stronie, UiPath zapisuje jego selektor — unikalny zestaw atrybutow, ktory pozwala go znalezc przy nastepnym uruchomieniu.

Typy selektorow w Modern Design
TypOpisKiedy uzywac
Strict SelectorDokladny selektor XML z konkretnymi atrybutami (id, name, class)Gdy element ma stabilne, unikalne atrybuty
Fuzzy SelectorSelektor oparty na "podobienstwie" — toleruje drobne zmianyGdy atrybuty moga sie lekko zmieniac
ImageRozpoznawanie wizualne (zrzut ekranu elementu)Ostatecznosc — gdy brak selektorow (np. Citrix, VDI)
Hierarchia niezawodnosci selektorow
Strict Selector (najlepszy) → Fuzzy Selector → Image (najgorszy)

Robot probuje w tej kolejnosci — jesli Strict nie zadziala, sprobuje Fuzzy, potem Image. Dlatego warto miec wszystkie trzy skonfigurowane.

Typowe atrybuty selektora
  • id — najbardziej stabilny (np. id='txtName')
  • name — czesto stabilny
  • class — zmienia sie przy aktualizacjach UI
  • idx (indeks) — najgorszy — zmienia sie gdy dodasz/usuniesz element na stronie
Zasada: Im bardziej "wewnetrzny" atrybut (id, name), tym stabilniejszy selektor. Unikaj idx i tableRow gdy to mozliwe.
Use Application/Browser — izolacja kontekstu

Use Application/Browser to kontener, ktory "podpina" robota do konkretnej aplikacji lub okna przegladarki. Wszystkie aktywnosci UI wewnatrz tego kontenera dzialaja w kontekscie tej aplikacji.

Dlaczego to wazne
  • Izolacja — robot wie, w ktorym oknie przegladarki klikac (wazne gdy masz wiele otwartych kart)
  • Zarzadzanie cyklem zycia — mozna skonfigurowac: otworz aplikacje na poczatku, zamknij na koncu
  • CloseMode — Always (zamknij po zakonczeniu), Never (zostaw otwarta), OnError (zamknij tylko przy bledzie)
Modul 2 — Czym jest zmienna?

Zmienna to "pudelko z etykieta". Na etykiecie jest nazwa (np. str_imie), pudelko ma okreslony rozmiar i ksztalt (typ danych — String, Integer, etc.), a w srodku jest zawartosc (wartosc).

Kluczowe cechy zmiennej
  • Nazwa — jak ja nazywamy (np. str_imie, int_wiek)
  • Typ danych — co moze przechowywac (tekst, liczbe, date...)
  • Scope — gdzie jest widoczna (w ktorym "pokoju" moze byc uzywana)
  • Wartosc domyslna — co jest w pudelku na poczatku (opcjonalnie)
Konwencja nazewnicza (zalecana)

Konwencja nazewnicza to umowa w zespole: "jak nazywamy zmienne, zeby kazdy wiedzial, co w nich jest, nawet bez czytania kodu". W UiPath najczesciej stosuje sie prefiksy wegierskie — trzy litery na poczatku nazwy wskazuja typ:

Scope — zakres widocznosci zmiennej

Scope to "pokoj", w ktorym zmienna jest widoczna. Zmienna zadeklarowana w Sequence jest widoczna tylko w tej Sequence i jej "dzieciach" (zagniezdonych kontenerach).

Main Sequence ← zmienna "A" (widoczna wszedzie ponizej)
  ├── Sequence 1 ← zmienna "B" (widoczna tylko w Seq 1)
  │     ├── Activity 1 ← widzi: A, B
  │     └── Activity 2 ← widzi: A, B
  └── Sequence 2 ← zmienna "C" (widoczna tylko w Seq 2)
        ├── Activity 3 ← widzi: A, C (NIE widzi B!)
        └── Activity 4 ← widzi: A, C
Zasady Scope
  1. Zmienna jest widoczna w Sequence, w ktorej jest zadeklarowana, i we wszystkich zagniezdonych kontenerach
  2. Zmienna NIE jest widoczna w kontenerach "obok" (sibling) ani "wyzej" (parent)
  3. Im wyzej w hierarchii zadeklarujesz zmienna, tym szersza widocznosc — ale nie przesadzaj! Deklaruj zmienna tam, gdzie jest naprawde potrzebna
Typowy blad: Zmienna zadeklarowana wewnatrz petli For Each — po wyjsciu z petli zmienna nie jest dostepna. Rozwiazanie: zadeklaruj ja na poziomie Sequence zawierajacej petle.

Konwersja typow — kiedy i dlaczego

UiPath (VB.NET) jest typowany statycznie — nie mozesz dodac String do Int32 bez konwersji. Typowe konwersje:

' String → Integer (gdy Input Dialog zwraca String z liczba)
cint("42")                    → 42

' String → Double (gdy potrzebujesz liczby zmiennoprzecinkowej)
cdbl("3.14")                  → 3.14

' Integer → String (gdy chcesz wyswietlic liczbe w tekscie)
42.ToString                   → "42"

' Integer → Double (gdy dzielisz dwie liczby calkowite)
cdbl(7) / cdbl(2)             → 3.5
' BEZ konwersji: 7 / 2 = 3 (obcina czesc dziesietna!)

' String → DateTime
DateTime.ParseExact("15-01-2024", "dd-MM-yyyy", Nothing)
Typowa pulapka — dzielenie Integer: 
int_wynik = 7 / 2    → 3  (nie 3.5! Integer obcina!)
dbl_wynik = cdbl(7) / cdbl(2)  → 3.5  (poprawnie!)

Split() to metoda typu String, ktora dzieli tekst na tablice (Array) na podstawie separatora:

"1,2,3,4".Split(",")        → {"1", "2", "3", "4"} ' dane CSV

Modul 3 — Czym jest DataTable?

DataTable to typ danych w UiPath (i .NET), ktory przechowuje dane tabelaryczne — wiersze i kolumny, dokladnie jak arkusz Excela. Roznica: DataTable istnieje w pamieci robota, a nie w pliku na dysku.

Dlaczego to wazne
  • Szybkosc — operacje na DataTable w pamieci sa wielokrotnie szybsze niz odczyt/zapis do pliku Excel przy kazdej operacji
  • Elastycznosc — mozesz filtrowac, sortowac, dodawac kolumny bez otwierania Excela
  • Wzorzec pracy: Wczytaj Excel → przetwarz w pamieci (DataTable) → zapisz wynik do nowego Excela
[Plik Excel] --Read Range--> [DataTable w pamieci] --operacje--> [DataTable] --Write Range--> [Nowy plik Excel]

Typowe pulapki przy pracy z DataTable
1. Typ Object

Dane wczytane z Excela przez Read Range sa typu Object, nie String czy Integer. Dlatego zawsze potrzebujesz .ToString lub cint():

' ZLE — Object nie ma metody Length:
row.Item("Imie").Length

' DOBRE — najpierw konwertuj:
row.Item("Imie").ToString.Length
2. Indeksowanie od 0

Pierwszy wiersz to Rows(0), pierwsza kolumna to Item(0):

dt.Rows(0).Item(0)   ' pierwszy wiersz, pierwsza kolumna
dt.Rows(1).Item(2)   ' drugi wiersz, trzecia kolumna
3. AddHeaders w Read Range

Jesli AddHeaders = True, pierwszy wiersz Excela staje sie nazwami kolumn, nie danymi. Jesli False, kolumny maja nazwy Column1, Column2...

Filter Data Table — szczegoly konfiguracji

Filter Data Table ma dwa panele:

1. Filter Rows — filtrowanie wierszy (warunki na wartosci)
  • Column: nazwa lub indeks kolumny
  • Operation: =, <>, >, <, >=, <=, Contains, StartsWith, EndsWith
  • Value: wartosc do porownania
  • Tryb: Keep (zostaw pasujace) lub Remove (usun pasujace)
2. Filter Columns — filtrowanie kolumn (ktore kolumny zachowac/usunac)
  • Tryb: Keep (zostaw wymienione) lub Remove (usun wymienione)
Wazne: Oba panele dzialaja jednoczesnie — mozesz filtrowac wiersze AND usuwac kolumny w jednym kroku.

Extract Table Data — jak to dziala

Extract Table Data (dawniej Data Scraping) to narzedzie do automatycznego wyodrebniania danych tabelarycznych ze stron www:

  1. Wskaz pierwszy element tabeli (np. pierwsza komorke)
  2. UiPath automatycznie rozpozna strukture tabeli
  3. Wskaz drugi element (w tym samym wierszu lub kolumnie) — UiPath potwierdzi wzorzec
  4. Konfiguruj kolumny (nazwy, typy, URL-e)
  5. Wynik: DataTable z wszystkimi danymi
Przydatne metody String
' .Split(" ").Last — wyciagnij ostatnie slowo:
"Tabela kursow z dnia 2024-01-15".Split(" ").Last  → "2024-01-15"

' .Split(" ").First — wyciagnij pierwsze slowo:
"Jan Kowalski".Split(" ").First  → "Jan"

' .Rows(0).Item(2) — konkretna komorka DataTable:
dt_euro.Rows(0).Item(2).ToString  → "4,5545"

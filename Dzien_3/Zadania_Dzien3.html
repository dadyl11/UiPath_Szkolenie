Moduł 1 — Pliki i foldery
Zadanie 1 — Kopiowanie pliku
Folder: Dzien3_Zadanie1_Pliki_Foldery/ · Temat: Operacje na plikach + Path.Combine + Path Exists

Opis zadania

  1. Pobierz bieżący katalog projektu (Get Environment Variable → CurrentDirectory)
  2. Użytkownik wybiera folder docelowy (Select Folder)
  3. Iteruj po plikach *.xlsx w bieżącym folderze (For Each File in Folder)
  4. Skopiuj plik do folderu docelowego (Copy File, Overwrite = True)
  5. Sprawdź czy skopiowany plik istnieje (Path Exists)
  6. Zaloguj wynik: "Czy skopiowano plik? " + bl_czy_skopiowano_plik.ToString

Wskazówki

  • Get Environment Variable z opcją CurrentDirectory — alternatywa dla hardkodowania ścieżek
  • Path.Combine(str_docelowy_folder, "Produkty.xlsx") — bezpieczne łączenie ścieżek
  • Path Exists — walidacja czy operacja się powiodła

Zadanie 5 — Logowanie z Orchestratora
Folder: Dzien3_Zadanie5_App_Logowanie/ · Temat: Get Credential + SecureString + Pick/PickBranch

Opis zadania

  1. Pobierz login i hasło: Get Credential → Asset: Robotivity_App_Credentials, FolderPath: Robotivity2
  2. Pobierz ścieżkę: Get Asset → Asset: Sciezka_Aplikacji, FolderPath: Robotivity2
  3. Zamknij Robotivity (Kill Process "robotivity")
  4. Otwórz Robotivity (Use Application/Browser)
  5. Wpisz login (Type Into → Text)
  6. Wpisz hasło (Type Into → SecureText!)
  7. Kliknij "LOG IN"
  8. Użyj aktywności Pick z dwoma PickBranch:
    • Branch 1 (Zalogowano): Trigger: Check App State na przycisk "EXIT" → Log: "Zalogowano do programu"
    • Branch 2 (Błędne hasło): Trigger: Check App State na tekst "The User name or password you enter is incorrect" → Click "OK" → Log (Warn): "Nie zalogowano do aplikacji. Sprawdz haslo"
Pick/PickBranch — aktywność "wyścigowa": pierwsza gałąź, której trigger się spełni, wygrywa. Używamy jej zamiast pojedynczego Check App State, bo mamy dwa możliwe scenariusze.
Uwaga: W Type Into dla hasła używaj właściwości SecureText (nie Text!) — hasło nigdy nie będzie widoczne w logach.

Zadanie 6 — Dispatcher
Folder: Dzien3_Zadanie6_Dispatcher/ · Temat: Bulk Add Queue Items + Add Queue Item z referencjami

Opis zadania

  1. Wczytaj Transactions.xlsx w Try Catch:
    • Try: Read Range Workbook (AddHeaders = True)
    • Catch: Kill Process "EXCEL" → Delay 2s → ponowny Read Range

Sposób 1 — Bulk Add (hurtowy, zakomentowany w projekcie):

  • Bulk Add Queue Items — QueueName: Robotivity_Bulk, FolderPath: Robotivity2

Sposób 2 — Add Queue Item z referencjami (aktywny):

  1. W pętli For Each Row:
    • Add Queue Item — QueueName: Robitivity_Single, FolderPath: Robotivity2
    • Reference: CurrentRow.Item("CashIn").ToString + "_" + CurrentRow.Item("OnUsCheck").ToString
    • ItemInformation: CashIn, OnUsCheck, NotOnUsCheck
  2. Log Message: "Transakcje dodane do kolejki"
Wskazówka: Try Catch wokół Read Range — Excel może blokować plik; w Catch: Kill Process EXCEL + Delay + ponów. Bulk Add jest szybszy, ale nie pozwala nadać indywidualnych referencji.

Zadanie 7 — Performer
Folder: Dzien3_Zadanie7_Performer/ · Temat: Flowchart + Queue + Robotivity + obsługa błędów

Opis zadania (Flowchart)

  1. Get Asset — pobierz Sciezka_Aplikacji z folderu Robotivity2
  2. Invoke Workflow File — Robotivity_App_Logowanie.xaml (argumenty: str_sciezka_aplikacji in, out_bool_zalogowano out)
  3. FlowDecision — out_bool_zalogowano?
    • False → "Koniec Pracy Robota" (If: zalogowano → log / Else → log "Nie zalogowano") + Kill Process "robotivity"
    • True → kontynuuj
  4. Pętla (strzałka powrotna):
    • Try Catch: Get Transaction Item z kolejki Robitivity_Single (FolderPath: Robotivity2)
      • Catch: qi_transaction_item = Nothing
    • FlowDecision — qi_transaction_item IsNot Nothing?
    • True → Log "Reference: " + Reference → Invoke Workflow File Procesowanie_Transakcji.xaml
    • W Procesowanie_Transakcji.xaml: wyciągnij dane, sprawdź .IsNumeric, Type Into + Click Accept lub Set Transaction Status = Failed
Wskazówka: Logowanie i procesowanie są wydzielone do osobnych plików .xaml (reużywalność). W Try Catch: gdy kolejka pusta → Exception → Catch: qi_transaction_item = Nothing → pętla się zakończy.

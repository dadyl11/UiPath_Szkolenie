Instrukcja — Konfiguracja Unattended Robot krok po kroku
Czym jest Unattended Robot?

Unattended Robot to robot UiPath, ktory dziala bez nadzoru uzytkownika — na maszynie, na ktorej nikt nie musi byc zalogowany. Procesy sa uruchamiane automatycznie z poziomu Orchestratora (harmonogram, trigger, API).

Wymagania wstepne
Maszyna z systemem Windows (Server lub Pro/Enterprise) z zainstalowanym UiPath Robot
UiPath Orchestrator — Community Cloud lub On-Premise
Licencja Unattended — przypisana do tenanta
Konto Windows na maszynie robota (lokalne lub domenowe) z uprawnieniami do logowania jako usluga
UiPath Robot zainstalowany w trybie Service Mode (nie User Mode)
Wazne: Robot w trybie User Mode dziala tylko gdy uzytkownik jest zalogowany. Dla Unattended wymagany jest Service Mode (instalacja jako usluga Windows).
Czesc 1 — UiPath Automation Cloud (Community / Enterprise Cloud)
Krok 1: Zaloguj sie do Orchestratora
  1. Wejdz na cloud.uipath.com
  2. Zaloguj sie na swoje konto
  3. Wybierz odpowiednia organizacje i tenanta
  4. Otworz Orchestrator
Krok 2: Dodaj Machine Template
Machine Template to szablon, ktory pozwala polaczyc jedna lub wiecej maszyn fizycznych z Orchestratorem.
  1. W Orchestratorze przejdz do: Tenant → Machines
  2. Kliknij Add Machine (lub +)
  3. Wybierz typ:
    • Standard Machine — jedna maszyna, jeden klucz
    • Machine Template — jeden szablon, wiele maszyn (zalecane)
  4. Podaj nazwe maszyny (np. ROBOT-SERVER-01)
  5. Skopiuj wygenerowany Machine Key — bedziesz go potrzebowal do polaczenia robota
Wskazowka: Machine Template jest zalecany — pozwala polaczyc wiele maszyn jednym szablonem i latwiej zarzadzac licencjami.
Krok 3: Polacz UiPath Robot z Orchestratorem
Na maszynie, na ktorej zainstalowales UiPath Robot.
  1. Otworz UiPath Assistant (ikona w zasobniku systemowym)
  2. Kliknij Preferences (zebatka) → Orchestrator Settings
  3. Wprowadz:
    • Orchestrator URL: https://cloud.uipath.com/{OrgName}/{TenantName}/orchestrator_
    • Machine Key: skopiowany klucz z kroku 2
  4. Kliknij Connect
  5. Status powinien zmienic sie na Connected, Licensed
Alternatywa — polaczenie przez wiersz polecen
cd "C:\Program Files\UiPath\Studio"
UiRobot.exe connect --url https://cloud.uipath.com/OrgName/TenantName/orchestrator_ --key MACHINE_KEY
Krok 4: Skonfiguruj Unattended Robot (konto)
  1. W Orchestratorze przejdz do: Tenant → Folders → wybierz folder (lub stworz nowy)
  2. Lub przejdz do: Tenant → Manage Access → Users
  3. Znajdz lub dodaj uzytkownika, ktory bedzie kontem robota
  4. Przy uzytkowniku kliknij Edit → zakladka Unattended Robot
  5. Wlacz przelacznik Enable Unattended Robot
  6. Wypelnij pola:
    • Domain\Username: konto Windows na maszynie robota (np. ROBOT-SERVER-01\RobotUser lub DOMENA\RobotUser)
    • Password: haslo do tego konta Windows
Wazne: To konto Windows musi istniec na maszynie i miec uprawnienie "Logowanie jako usluga" (Log on as a service). Robot uzyje tych danych do utworzenia sesji Windows na maszynie.
Krok 5: Przypisz robota do folderu
  1. W Orchestratorze przejdz do wybranego folderu
  2. Settings → Machines — przypisz Machine Template z kroku 2
  3. Settings → Users — przypisz uzytkownika z kontem Unattended z kroku 4
Krok 6: Opublikuj proces i stworz harmonogram
  1. Z UiPath Studio: Publish → wybierz Orchestrator jako cel
  2. W Orchestratorze: Automations → Processes → Add Process → wybierz pakiet
  3. Automations → Triggers → Add Trigger:
    • Wybierz proces
    • Ustaw harmonogram (CRON lub kalendarz)
    • Wybierz konto/maszyne do uruchomienia
  4. Kliknij Save
Krok 7: Weryfikacja
  1. Monitoring → Jobs — sprawdz czy zadania sie uruchamiaja
  2. Status maszyny: Tenant → Machines — powinna byc "Available"
  3. Monitoring → Robots — robot powinien miec status "Available"
Czesc 2 — UiPath Orchestrator On-Premise
Krok 1: Zaloguj sie do Orchestratora
  1. Otworz przegladarke i wejdz na adres swojego Orchestratora (np. https://orchestrator.firma.pl)
  2. Zaloguj sie danymi administratora
Krok 2: Sprawdz licencje
  1. Przejdz do: Tenant → License
  2. Upewnij sie, ze masz dostepne licencje Unattended
  3. Jesli nie — zaimportuj plik licencji lub aktywuj online
Krok 3: Dodaj maszyne
  1. Przejdz do: Tenant → Machines
  2. Kliknij + (Add)
  3. Wybierz Standard Machine lub Machine Template
  4. Podaj nazwe maszyny — musi odpowiadac nazwie hosta (hostname) maszyny fizycznej
    • Sprawdz hostname: na maszynie robota uruchom hostname w CMD
  5. Skopiuj Machine Key
Uwaga On-Premise: W wersji On-Premise nazwa Standard Machine MUSI dokladnie odpowiadac hostname maszyny. W Machine Template nie ma tego ograniczenia.
Krok 4: Zainstaluj i polacz UiPath Robot
Instalacja w trybie Service Mode
  1. Na maszynie robota zainstaluj UiPath Robot w trybie Service Mode
  2. Podczas instalacji wybierz: "Service Mode" (nie User Mode)
Instalacja przez wiersz polecen
UiPathStudio.msi ADDLOCAL=DesktopFeature,Robot,RegisterService INSTALL_TYPE=ServiceMode
Polaczenie z Orchestratorem
  1. Otworz UiPath Assistant → Preferences → Orchestrator Settings
  2. Orchestrator URL: https://orchestrator.firma.pl
  3. Machine Key: klucz z kroku 3
  4. Kliknij Connect
Krok 5: Stworz Unattended Robot
Metoda nowoczesna (wersja 2021.10+)
  1. Tenant → Manage Access → Users → dodaj uzytkownika lub znajdz istniejacego
  2. Edytuj uzytkownika → zakladka Unattended Robot
  3. Wlacz Enable Unattended Robot
  4. Podaj Domain\Username i Password konta Windows
Metoda klasyczna (starsze wersje Orchestratora)
  1. Przejdz do: Tenant → Robots
  2. Kliknij + (Add) → Standard Robot
  3. Wypelnij:
    • Machine: wybierz maszyne z kroku 3
    • Name: nazwa robota (dowolna)
    • Domain\Username: konto Windows (np. SERWER\RobotSvc)
    • Password: haslo konta Windows
    • Type: Unattended
  4. Kliknij Create
Krok 6: Przypisz do folderu i stworz proces

Identycznie jak w czesci Cloud (Krok 5 i 6 z czesci 1):

  1. Przypisz maszyne i uzytkownika do folderu
  2. Opublikuj proces z UiPath Studio
  3. Stworz Process i Trigger w Orchestratorze
Krok 7: Konfiguracja konta Windows

Na maszynie robota upewnij sie, ze konto ma odpowiednie uprawnienia:

  1. Otworz: secpol.msc (Lokalna polityka zabezpieczen)
  2. Przejdz do: Local Policies → User Rights Assignment
  3. Znajdz: "Log on as a service" (Logowanie jako usluga) → dodaj konto robota
  4. Znajdz: "Allow log on locally" (Zezwalaj na logowanie lokalne) → dodaj konto robota
  5. Jesli uzywasz Remote Desktop:
    • Znajdz: "Allow log on through Remote Desktop Services" → dodaj konto robota
Wazne: Jesli maszyna jest w domenie Active Directory, te ustawienia moga byc nadpisywane przez Group Policy (GPO). Skontaktuj sie z administratorem domeny.

Dobre praktyki
Dedykowane konto — stworz osobne konto Windows tylko dla robota (np. svc_uipath_robot)
Haslo bez wygasania — ustaw "Password never expires" dla konta robota (lub zarzadzaj przez Credential Store)
Service Mode — zawsze instaluj robota w trybie Service Mode dla Unattended
Machine Template — uzywaj Machine Template zamiast Standard Machine dla latwiejszego skalowania
Credential Store — przechowuj hasla w Orchestrator Credential Store zamiast wpisywac je recznie
Monitoring — skonfiguruj alerty w Orchestratorze (Alerts) na wypadek bledow
Rozdzielczosc ekranu — ustaw stala rozdzielczosc (np. 1920x1080) dla sesji robota, zeby selektory UI dzialaly poprawnie
High-DPI — wylacz skalowanie DPI (ustaw na 100%) na maszynie robota
Windows Update — zaplanuj aktualizacje poza godzinami pracy robota
Logi — monitoruj logi w Orchestratorze (Monitoring → Logs) i konfiguruj retencje
Schemat konfiguracji — podsumowanie
1. ORCHESTRATOR
   ├── Dodaj Machine Template / Standard Machine
   ├── Skopiuj Machine Key
   ├── Dodaj/edytuj uzytkownika
   │   └── Wlacz Unattended Robot (Domain\Username + Password)
   └── Przypisz maszyne + uzytkownika do folderu

2. MASZYNA ROBOTA
   ├── Zainstaluj UiPath Robot (Service Mode)
   ├── Polacz z Orchestratorem (URL + Machine Key)
   ├── Skonfiguruj konto Windows (Log on as service)
   └── Ustaw rozdzielczosc + wylacz DPI scaling

3. URUCHOMIENIE
   ├── Opublikuj proces z UiPath Studio
   ├── Stworz Process w Orchestratorze
   ├── Dodaj Trigger (harmonogram / kolejka)
   └── Monitoruj w Jobs / Logs
